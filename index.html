how does this work? to encode/decode a TOTP key:
  1. i hash the password with SHA1
  2. i XOR the bits of the encoded (or decoded - XOR works both ways) key with the hash; the hash is twice as long as the TOTP key so i decided to use both halves of the hash on the TOTP key
  3. use the XOR'd key as the new TOTP key
in theory this is secure because there's no way to tell if a TOTP key is correct other than to try it against github, which ratelimits a ton. also, assuming the password to the TOTP key isn't super vulnerable to a dictionary attack (trust me, it isn't), guessing passwords is no better than guessing 6-digit codes at random.
